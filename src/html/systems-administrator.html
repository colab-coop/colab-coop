Systems Administrator

The CoLab community is looking to welcome a unique soul who is passionate about collaboration, loves diving deep into the configuration of complex web systems, and understands what web developers and site owners need. As a cooperatively owned and managed company, we are dedicated to mutual support, deeply democratic decision making and full-hearted communication. If that makes you happy, we’d love to talk.

Our systems administrators are best-prepared by at least 2 years of experience administering web systems and DevOps tools, as well as some form of relevant schooling, training or equivalent work experience. Because many of our tools are open source, involvement in some open source project will also be helpful.

Experience

We work with a range of tools and platforms, and while all of them are not required, experience with some of the following technologies is important to take on the tasks you’ll be responsible for:

  • Linux systems administration: Ubuntu, CentOS, Amazon Linux.
  • NodeJS: setting up servers running NodeJS apps and API servers, including administration of web server technologies such as nginx, haproxy, pm2, etc.
  • PHP web server administration for common platforms such as Drupal, Wordpress and MVC frameworks like Yii or CodeIgniter.
  • Administration of a range of Amazon Web Services, including EC2 clusters, ELBs, Route 53 DNS, etc.
  • DNS management using Route 53, DNS Made Easy, and a range of DNS services our clients choose to work with that we’d rather not name.
  • Administration of database systems, including MySQL and a Mongo cluster or two.
  • Experience with Puppet is a plus — we use it to manage our development environment skeletons and production site configurations.
  • Familiarity with OS X developer environment setup via Homebrew, Puppet, etc.
  • Familiarity with Atlassian products such as JIRA and Bitbucket, which we use to coordinate all projects.
  • Git experience and knowledge is a must, and be prepared to memorize the git flow workflow is you haven’t already.
  • Systems monitoring via Sensu for tracking key system health indicators.
What you’ll be doing

As a cooperatively-run company, all roles at CoLab evolve and develop over time. As a systems administrator on our team, you can expect to be responsible for some or all of the following tasks initially:

  • Initializing new projects systems: including setting up git repositories, chat-rooms, email lists and JIRA agile projects.
  • Setting up development environments for new projects: using Puppet to quickly bootstrap development servers for staging and development of NodeJS APIs, Drupal sites, Django stacks, etc.
  • Systems Intake for New Clients: review and documentation of new clients’ hosting systems, DNS and registration accounts, integrated 3rd party systems, etc.
  • Ongoing development support: support for developers with systems updates, Git issues, etc.
  • Production Systems Monitoring: set up and monitoring of key systems metrics and execution of key maintenance tasks called for my monitor results.
  • Backup Configuration and Administration: set up backup systems, monitoring of backups and periodic testing.
  • Launch and Deployment Support: set up of deployment environments, coordination with PM and dev team regarding systems tasks, pro-active planning of key launch needs, launch window availability and support.
  • Systems Documentation: documentation of key client systems architectures, technical administration processes and secure credential storage.
  • Systems Reporting: support IT heads with monthly report data on systems uptime, systems health and anticipated costs and cost reduction options.
What to expect

This is a contract position. While there are no set hours, we do require candidates who are available for some meetings and calls during the bulk of US working hours and can commit to timely communication around in-progress items with developers based in the US, Europe and Asia.

We anticipate the job will involve 10-20 hours of work initially, as the systems are configured, for a period of one to two months. Following this the role will likely transition into process improvement and systems support, which may see either an increase or a decrease in hours depending on context.
